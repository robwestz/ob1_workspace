# 2026-02-03

## Session Notes
- Started Bacowr v6.0 -- SEO article pipeline concept crystallized
- Defined the 8-phase pipeline: preflight -> publisher profiling -> SERP analysis -> semantic bridge -> blueprint -> write -> QA -> output
- Key decision: use Anthropic (Claude) for article writing, OpenAI for embeddings
- First test articles showed promise but QA was way too lenient -- articles passed that shouldn't have
- Named it Bacowr: Bridge-Anchored Content Orchestration with Research

## Decisions
- Pipeline architecture: pipeline.py (preprocessing + orchestration) + engine.py (generation core)
- 11-point QA gate -- every article must pass all 11 checks before output
- QA checks include: word count, anchor placement, keyword density, readability, factual grounding, bridge coherence, publisher voice match, no AI tells, internal linking, meta description, uniqueness score
- Separate models for separate tasks: Claude writes, OpenAI embeds, potentially Gemini for large-context research

## Technical Notes
- Core files: engine.py (~2800 lines), pipeline.py (~1200 lines), models.py, article_validator.py
- Total: ~5,008 lines Python
- Input: CSV with columns (publisher_domain, target_url, anchor_text, keywords)
- Output: formatted articles with metadata JSON
