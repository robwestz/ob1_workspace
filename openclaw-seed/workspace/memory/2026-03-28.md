# 2026-03-28

## Session Notes
- First real overnight session experiment — 4 waves over ~3 hours
- Wave protocol tested: Plan -> Execute -> Verify -> Fix -> Commit -> Assess
- Key failure mode discovered: batch-and-done kills the session early
- Key success mode: iterative waves where each wave responds to what the previous wave found

## What Worked
- Wave structure kept the agent focused — no drift into random exploration
- VERIFY step caught 3 regressions that would have gone unnoticed in a batch approach
- COMMIT per wave meant we had checkpoints even when wave 4 hit a blocker
- Morning report updated after each wave — even a crashed session produces useful output

## What Failed
- First attempt was "launch 4 parallel agents, commit, done" — finished in 25 minutes
- That's not a night shift, that's a sprint. We lost 7+ hours of potential work.
- Agent couldn't figure out what to do next without ASSESS step guidance
- Budget tracking was missing — no idea what we spent until checking the console

## Lessons Learned
1. Night shifts must be ITERATIVE, not batch. Plan, do, check, adapt, repeat.
2. Each wave's results inform the next wave's plan — this is where the intelligence lives
3. Budget tracking needs to be built into the runtime, not checked manually
4. The agent needs self-direction heuristics: "what's the highest-value next wave?"
5. 3 strikes rule: if something fails verification 3 times, document it and move on
6. Push every 2-3 waves, not just at the end — crash = lost work otherwise
