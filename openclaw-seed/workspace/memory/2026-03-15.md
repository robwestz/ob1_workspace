## The Harness Idea
Traditional repos have READMEs for humans. Agent-first repos need AGENTS.md for AI. The harness is a set of constraints, knowledge maps, and quality gates that make AI coding agents effective in this specific codebase. It's not about limiting the agent — it's about giving it the context to make good decisions autonomously.

## Harness Files Created
- .harness/principles.yml — 8 core engineering principles
- .harness/quality.yml — quality gate definitions (compile, test, lint, build, size)
- .harness/domains.yml — domain boundaries and ownership
- .harness/enforcement.yml — what's enforced automatically vs. manually reviewed
- .harness/config.yml — harness configuration
- .harness/knowledge.yml — knowledge map pointers

## Key Principle: "Fix broken things first"
When an agent starts a session, first thing it does: run tests. If anything is red, that's the work. Don't add features on top of a broken foundation. This seems obvious but every agent system I've seen skips it.
